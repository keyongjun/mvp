# PRD: 개인 개발 블로그 (Notion CMS)

## 1. 프로젝트 개요

| 항목       | 내용                                   |
| ---------- | -------------------------------------- |
| 프로젝트명 | 개인 개발 블로그                       |
| 목적       | Notion을 CMS로 활용한 개인 기술 블로그 |
| 배포 환경  | Vercel                                 |

### 배경 및 목적

Notion에서 글을 작성하면 자동으로 블로그에 반영되는 구조로, 별도의 CMS 관리 없이 Notion 워크스페이스만으로 콘텐츠를 관리한다. 개발자가 익숙한 Notion 환경에서 글을 작성하면 즉시 블로그에 게시되어 콘텐츠 관리 비용을 최소화한다.

---

## 2. 기술 스택

| 분류       | 기술                            |
| ---------- | ------------------------------- |
| Frontend   | Next.js 15, TypeScript          |
| CMS        | Notion API (`@notionhq/client`) |
| Styling    | Tailwind CSS, shadcn/ui         |
| Icons      | Lucide React                    |
| Deployment | Vercel                          |

---

## 3. Notion 데이터베이스 구조

블로그 글은 Notion 데이터베이스의 각 페이지로 관리된다.

| 필드명    | 타입         | 설명                                         |
| --------- | ------------ | -------------------------------------------- |
| Title     | title        | 글 제목                                      |
| Category  | select       | 카테고리 (예: React, Next.js, TypeScript 등) |
| Tags      | multi_select | 태그 (복수 선택 가능)                        |
| Published | date         | 발행일                                       |
| Status    | select       | 상태 (`초안` / `발행됨`)                     |
| Content   | page content | 본문 (Notion 페이지 블록)                    |

> **참고:** `Status`가 `발행됨`인 글만 블로그에 노출된다.

---

## 4. 주요 기능

### 4.1 글 목록 조회

- Notion 데이터베이스에서 `Status = 발행됨`인 글 목록을 가져온다.
- 발행일 기준 최신순으로 정렬한다.
- 제목, 카테고리, 태그, 발행일을 카드 형태로 표시한다.

### 4.2 글 상세 페이지

- 개별 글의 Notion 페이지 블록을 파싱하여 렌더링한다.
- 지원 블록 타입: 단락, 제목(H1~H3), 글머리 기호, 번호 목록, 코드 블록, 인용, 이미지, 구분선.

### 4.3 카테고리별 필터링

- 카테고리 선택 시 해당 카테고리의 글 목록만 표시한다.
- URL 파라미터(`?category=React`)로 상태를 관리하여 공유 가능한 링크를 제공한다.

### 4.4 검색 기능

- 제목 및 태그 기반 클라이언트 사이드 검색을 지원한다.
- 검색어 입력 시 실시간으로 결과를 필터링한다.

### 4.5 반응형 디자인

- 모바일(320px~), 태블릿(768px~), 데스크톱(1024px~) 뷰포트를 지원한다.
- Tailwind CSS 반응형 유틸리티 클래스를 활용한다.

## 5. 화면 구성

### 5.1 홈 (`/`)

- 최근 발행된 글 목록 표시 (카드 레이아웃)
- 카테고리 필터 탭
- 검색 입력창

```
┌─────────────────────────────────┐
│  [로고/블로그명]        [검색창] │
├─────────────────────────────────┤
│  [전체] [React] [Next.js] [TS]  │  ← 카테고리 탭
├─────────────────────────────────┤
│  ┌──────┐  ┌──────┐  ┌──────┐  │
│  │ 글1  │  │ 글2  │  │ 글3  │  │  ← 글 카드
│  └──────┘  └──────┘  └──────┘  │
└─────────────────────────────────┘
```

### 5.2 글 상세 (`/posts/[slug]`)

- 제목, 발행일, 카테고리, 태그 표시
- Notion 블록 기반 본문 렌더링
- 목록으로 돌아가기 링크

```
┌─────────────────────────────────┐
│  ← 목록으로                      │
│                                 │
│  [글 제목]                       │
│  2026-04-05 · React · #hooks    │
├─────────────────────────────────┤
│  본문 내용...                    │
│                                 │
└─────────────────────────────────┘
```

### 5.3 카테고리 (`/category/[name]`)

- 선택된 카테고리의 글 목록 표시
- 홈과 동일한 카드 레이아웃

## 7. 구현 단계

### 1단계: 환경 설정

- `@notionhq/client` 패키지 설치
- `.env.local`에 `NOTION_API_KEY`, `NOTION_DATABASE_ID` 설정
- Notion Integration 생성 및 데이터베이스 공유

### 2단계: Notion 데이터베이스 구성

- Notion에서 블로그용 데이터베이스 생성
- 위 스키마에 맞게 속성 추가
- 테스트용 샘플 글 작성 (Status: 발행됨)

### 3단계: 글 목록 페이지 구현

- Notion API로 데이터베이스 쿼리하는 서버 함수 작성
- 홈 페이지(`/`) 글 카드 목록 렌더링
- 카테고리 필터 및 검색 구현

### 4단계: 글 상세 페이지 구현

- Notion 페이지 블록을 가져오는 함수 작성
- 블록 타입별 렌더링 컴포넌트 구현
- 동적 라우트(`/posts/[slug]`) 설정

### 5단계: 스타일링 및 최적화

- shadcn/ui 컴포넌트 적용
- 반응형 레이아웃 완성
- Next.js ISR(Incremental Static Regeneration) 설정으로 성능 최적화

## 8. API 설계

### Notion 유틸리티 함수

| 함수명                         | 설명                     |
| ------------------------------ | ------------------------ |
| `getPosts()`                   | 발행된 글 전체 목록 조회 |
| `getPostBySlug(slug)`          | 슬러그로 특정 글 조회    |
| `getPostsByCategory(category)` | 카테고리별 글 목록 조회  |
| `getPageBlocks(pageId)`        | 글 본문 블록 조회        |
